# 기본형 vs 참조형 1 - 시작

자바에서 **참조형(Reference Type)** 을 제대로 이해하는 것은 정말 중요하다.
지금까지 기본형과 참조형에 대해서 조금씩 보았다면, 이제 두 개념을 **조금 더 깊이 있게 정리**해 보자.

자바에서 변수의 데이터 타입은 크게 다음 두 가지로 나눌 수 있다.

* **기본형(Primitive Type)**
* **참조형(Reference Type)**

이 둘의 차이를 이해하면, 이후에 **객체, 배열, 컬렉션** 등을 공부할 때 훨씬 수월해진다.

---

## 1. 기본형과 참조형의 큰 그림

먼저 변수에 어떤 값이 **어떻게 저장되는지**를 기준으로 나누어 보자.

* **기본형(Primitive Type)**
  `int`, `long`, `double`, `boolean` 처럼
  **변수 안에 사용할 값을 직접 넣어 두는 타입**을 말한다.

* **참조형(Reference Type)**
  `Student student1`, `int[] students` 와 같이
  **데이터에 접근하기 위한 참조(주소)를 저장하는 타입**을 말한다.
  참조형은 **객체** 또는 **배열**에 사용된다.

> 쉽게 이야기해서
>
> * **기본형 변수** 안에는 곧바로 사용할 수 있는 “값”이 들어 있고,
> * **참조형 변수** 안에는 “어디에 값이 있는지 가리키는 위치(참조값)”이 들어 있다.

참조형 변수를 통해 무언가를 사용하려면,
**참조값을 따라가서 그 위치에 있는 실제 데이터에 접근**해야 한다.

---

## 2. 기본형 vs 참조형 - 기본

### 2.1 기본형(Primitive Type)의 특징

기본형은 예를 들어 다음과 같이 사용할 수 있다.

* `int`
* `long`
* `double`
* `boolean`

이러한 타입의 변수에는 **숫자 10, 20 같은 실제 값**이 직접 들어간다.
그래서 이 값을 **바로 읽어서 사용**할 수 있다.

> 정리
>
> * 값이 변수 안에 **직접 저장**된다.
> * 값을 바로 꺼내서 **연산에 그대로 사용**할 수 있다.

### 2.2 참조형(Reference Type)의 특징

참조형은 실제 사용하는 값을 변수에 직접 담지 않는다.
**변수 안에는 “객체가 어디에 있는지”에 대한 참조값(주소)** 만 들어 있다.

* 예: `Student student1`, `int[] students`
* 이 변수들에는 **객체나 배열이 있는 메모리 위치(주소)** 가 들어 있다.

참조형에는 크게 두 가지 대상이 있다.

* **객체(Object)**

    * `.`(dot) 연산자를 통해
      메모리 상에 생성된 객체를 찾아가서 사용한다.
* **배열(Array)**

    * `[]`(대괄호, 인덱스)를 통해
      메모리 상에 생성된 배열을 찾아가서 사용한다.

---

## 3. 기본형 vs 참조형 - 계산(연산)

이제 **연산 관점**에서 차이를 보자.

### 3.1 기본형은 값을 그대로 연산에 사용

기본형은 변수 안에 들어 있는 값을 **그대로** 계산에 사용할 수 있다.
(예: 더하기, 빼기 등)

```java
int a = 10;
int b = 20;
int sum = a + b;
```

* `a` 와 `b` 에는 각각 `10`, `20` 이라는 값이 **직접** 들어 있다.
* 따라서 `a + b` 와 같이 바로 **산술 연산**에 사용할 수 있다.

> 기본형은 변수에 실제 값이 들어 있기 때문에
> `+`, `-` 와 같은 연산이 가능하다.

### 3.2 참조형은 “참조값” 자체로는 연산 불가

참조형은 변수 안에 **객체의 위치(참조값)** 이 들어 있다.
이 **참조값 자체는 숫자 연산에 사용할 수 없다.**

```java
Student s1 = new Student();
Student s2 = new Student();
s1 + s2 //오류 발생
```

* `s1`, `s2` 에는 `Student` 객체의 **참조값(주소)** 가 들어 있다.
* `s1 + s2` 처럼 **참조값끼리 더하려고 하면 연산이 불가능**하므로 오류가 발생한다.

즉, **주소지만 가지고는 할 수 있는 게 없다.**
**주소가 가리키는 곳에 있는 실제 값**을 사용해야 한다.

> 정리
>
> * 기본형은 **연산 가능**하다.
> * 참조형은 **참조값만 가지고는 연산할 수 없다.**

### 3.3 참조형도 “내부의 기본형 값”은 연산 가능

참조형의 변수 자체는 연산할 수 없지만,
**그 변수가 가리키는 객체(또는 배열) 안의 기본형 값**은 연산할 수 있다.

```java
Student s1 = new Student();
s1.grade = 100;

Student s2 = new Student();
s2.grade = 90;

int sum = s1.grade + s2.grade; //연산 가능
```

* `s1`, `s2` 는 `Student` 객체의 **참조값**이다.
* `s1.grade`, `s2.grade` 는 각각 **기본형(예: int) 변수**다.
* 따라서 `s1.grade + s2.grade` 처럼
  **기본형 값끼리의 연산은 정상적으로 가능**하다.

결국 참조형을 사용할 때는:

1. 참조형 변수(`s1`, `s2`)로 **객체의 위치**를 찾고
2. `.` 또는 `[]` 를 사용해서 **객체/배열 내부의 기본형 값**에 접근한 다음
3. 그 **기본형 값들을 가지고 계산**하게 된다.

## 4. 쉽게 이해하는 팁

기본형과 참조형을 구분할 때 다음 내용을 기억해 두면 편하다.

* **기본형을 제외한 나머지는 모두 참조형이다.**
* **기본형 타입 이름은 소문자로 시작**한다.

    * 예: `int`, `long`, `double`, `boolean`
* **기본형은 자바가 기본으로 제공하는 데이터 타입**이다.

    * 개발자가 **직접 새로운 기본형을 정의할 수는 없다.**
* 개발자가 새로 정의할 수 있는 것은 **클래스**이고,
  이 클래스들은 모두 **참조형**이다.
* **클래스 이름은 관례적으로 대문자로 시작**한다.

    * 예: `Student` 클래스는 **참조형**이다.

---

## 5. 참고 - String

자바에서 `String` 은 조금 특별한 존재다.

* `String` 은 사실 **클래스**이므로, 원칙적으로는 **참조형**이다.
* 그런데 다음과 같이 **기본형처럼 문자 값을 바로 대입**할 수 있다.

```java
String name = "hello";
```

문자열은 자바에서 **아주 자주 사용하는 타입**이기 때문에,
자바는 `String` 에 대해 **편의 기능을 특별히 제공**한다.

`String` 의 자세한 내용은 뒤에서 별도로 더 다루게 된다.
