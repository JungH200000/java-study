# 정리: 기본형 vs 참조형

## 1. 대원칙 – 자바는 **항상 변수의 값을 복사해서** 대입한다

* 자바에서 변수에 값을 대입한다는 것은,
  **그 변수 안에 들어 있는 값을 읽어서 그대로 복사해 넣는 것**이다.
* 이 원칙은 **기본형, 참조형 모두 동일**하다.

    * 기본형 → 변수에 들어 있는 **실제 사용하는 값**을 복사해서 대입
    * 참조형 → 변수에 들어 있는 **참조값(주소)** 을 복사해서 대입
* 따라서 **“기본형이든 참조형이든, 대입 시에는 변수의 값을 복사한다”** 는 방식은 같지만,
  **그 값의 종류(실제 값 vs 참조값)** 에 따라 동작 결과는 달라진다.

---

## 2. 기본형 vs 참조형 – 기본

자바의 데이터 타입을 크게 나누면 다음과 같다.

* **기본형(Primitive Type)**
* **참조형(Reference Type)**

여기서:

* **기본형을 제외한 나머지 변수는 모두 참조형**이다.

    * 예: **클래스를 다루는 변수, 배열을 다루는 변수**는 참조형이다.

두 타입의 차이:

* **저장하는 값**

    * 기본형 변수: **값 자체를 직접 저장**
    * 참조형 변수: **참조(주소)를 저장**
* **연산 가능 여부**

    * 기본형 변수: **산술 연산을 수행할 수 있다.**
    * 참조형 변수: **산술 연산을 수행할 수 없다.**
* **null 할당 가능 여부**

    * 기본형 변수: **`null` 을 할당할 수 없다.**
    * 참조형 변수: **`null` 을 할당할 수 있다.**

---

## 3. 기본형 vs 참조형 – 대입

* 기본형과 참조형 모두, **대입 시에는 변수 안에 있는 값을 읽어서 복사해 전달**한다.

    * 기본형: **실제 사용하는 값**을 복사해서 전달
    * 참조형: **참조값(주소)** 을 복사해서 전달

여기서 중요한 점:

* **실제 인스턴스(객체)가 복사되는 것이 아니다.**
* **인스턴스를 가리키는 참조값을 복사해서 전달**하는 것이다.
* 이 때문에 **하나의 인스턴스를 여러 곳에서 함께 참조할 수 있다.**

헷갈릴 때는 이렇게 생각하면 된다.

> “변수 안에 무엇이 들어있는지를 떠올리자.
> 기본형은 ‘사용하는 값’이, 참조형은 ‘참조값’이 들어 있다.
> **어떤 값이 들어있든, 대입할 때는 그 값을 그대로 복사해서 전달한다.**”

---

## 4. 기본형 vs 참조형 – 메서드 호출

메서드 호출에서도 **“값을 복사해 전달한다”는 원칙은 같다.**
하지만 전달되는 값이 **실제 값인지, 참조값인지**에 따라 결과가 달라진다.

* **기본형**

    * 메서드로 기본형 데이터를 전달하면,
      **값이 복사되어 매개변수(파라미터)에 들어간다.**
    * 메서드 내부에서 이 매개변수의 값을 변경해도,
      **호출한 쪽 변수의 값에는 영향이 없다.**

* **참조형**

    * 메서드로 참조형 데이터를 전달하면,
      **참조값이 복사되어 매개변수에 들어간다.**
    * 메서드 내부에서 매개변수(파라미터)를 통해 **전달된 객체의 멤버 변수 값을 변경하면**,
      **호출한 쪽에서 참조하는 객체도 함께 변경**된다.
