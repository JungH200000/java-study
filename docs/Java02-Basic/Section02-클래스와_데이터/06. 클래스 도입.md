# 클래스 도입

앞서 살펴본 학생 정보 문제를 **클래스**를 도입해서 해결해 보자.
클래스를 사용하면 **학생이라는 개념**을 하나로 만들고,
각 학생마다 **자신의 이름, 나이, 성적**을 함께 관리할 수 있다.

우선 학생을 표현하는 클래스를 만들고, 이어서 그 클래스를 사용하는 코드를 보자.

---

## 1. `Student` 클래스 정의

### 1-1. 코드

```java
package class1;

public class Student {
    String name;
    int age;
    int grade;
}
```

### 1-2. 설명

* `class` 키워드를 사용해서 **학생 클래스(`Student`)** 를 정의했다.
* `Student` 클래스 내부에는 다음과 같은 변수가 있다.

    * 이름: `String name`
    * 나이: `int age`
    * 성적: `int grade`
* 이렇게 **클래스 안에 정의된 변수들**을

    * **멤버 변수(Member Variable)** 또는
    * **필드(Field)**
      라고 부른다.

#### 멤버 변수 / 필드

* **멤버 변수(Member Variable)**

    * 특정 클래스에 소속된 **멤버(구성원)** 이라는 의미에서 이렇게 부른다.
* **필드(Field)**

    * 데이터 항목을 가리키는 전통적인 용어이다.
      데이터베이스, 엑셀 등에서 각각의 데이터 항목을 **필드**라고 한다.
* 자바에서는 **멤버 변수와 필드가 같은 뜻**이다.
  → 둘 다 **클래스에 소속된 변수**를 의미한다.

#### 클래스 이름 규칙

* 클래스 이름은 **관례상 대문자로 시작**한다.
* 단어가 여러 개일 때는 **낙타 표기법(카멜 케이스)** 을 사용한다.

    * 예: `Student`, `User`, `MemberService`

## 2. `Student` 클래스를 사용하는 코드 – `ClassStart3`

이제 방금 만든 **학생 클래스(`Student`)를 실제로 사용하는 코드**를 보자.

### 2-1. 코드: `ClassStart3`

```java
package class1;

public class ClassStart3 {
    public static void main(String[] args) {
        Student student1;
        student1 = new Student();
        student1.name = "학생1";
        student1.age = 15;
        student1.grade = 90;

        Student student2 = new Student();
        student2.name = "학생2";
        student2.age = 16;
        student2.grade = 80;

        System.out.println("이름:" + student1.name + " 나이:" + student1.age + " 성적:" + student1.grade);
        System.out.println("이름:" + student2.name + " 나이:" + student2.age + " 성적:" + student2.grade);
    }
}
```

### 2-2. 실행 결과

```text
이름:학생1 나이:15 성적:90
이름:학생2 나이:16 성적:80
```

## 3. 클래스와 사용자 정의 타입

### 3-1. 타입이란?

* **타입(type)** 은 데이터의 **종류나 형태**를 나타낸다.

    * `int` → 정수 타입
    * `String` → 문자(문자열) 타입

그렇다면 **학생을 나타내는 타입**이 필요할 때는 어떻게 할까?
→ `Student` 라는 **새로운 타입**을 만들면 된다.

### 3-2. 클래스로 타입 만들기

* 클래스를 사용하면 `int`, `String` 처럼 **직접 타입을 만들 수 있다.**
* 사용자가 직접 정의하는 이런 타입을 **사용자 정의 타입**이라고 한다.
* 사용자 정의 타입을 만들려면 **설계도**가 필요한데,
  이 설계도가 바로 **클래스**이다.

### 3-3. 클래스와 객체(인스턴스)

* 설계도인 **클래스**를 사용해서 실제 메모리에 만들어진 실체를
  **객체(object)** 또는 **인스턴스(instance)** 라고 부른다.
* 클래스를 통해 사용자가 원하는 종류의 데이터 타입을 **마음껏 정의**할 수 있다.

---

## 4. 용어 정리: 클래스, 객체, 인스턴스

* **클래스(Class)**

    * 객체를 만들기 위한 **설계도**이다.
* **객체(Object) / 인스턴스(Instance)**

    * 클래스를 기반으로 **실제 메모리에 만들어진 실체**이다.
    * 두 용어는 **같은 의미**로 사용된다.

예를 들어, 이 글에서는:

* `Student` → **학생을 표현하는 클래스(설계도)**
* `student1`, `student2` → `Student` 클래스를 기반으로 만든
  **학생1, 학생2 객체(인스턴스)** 이다.

---

## 5. 코드 한 줄씩 분석하기

이제 `ClassStart3` 코드의 중요한 부분을 단계별로 살펴보자.

### 5-1. 1단계: 변수 선언

<img src="./images/06/01VariableDeclaration.png" width=450px>

```java
Student student1;
```

* `Student student1`
  → `Student` 타입을 받을 수 있는 변수를 선언한다.
* `int`가 정수를, `String`이 문자를 담을 수 있듯이,
  **`Student`는 `Student` 타입의 객체(인스턴스)를 담을 수 있다.**

### 5-2. 2단계: 객체 생성

<img src="./images/06/02CreateObject.png" width=600px>

```java
student1 = new Student();
```

이 코드를 나누어 이해하면 다음과 같다.

* 객체를 사용하려면 먼저 **설계도인 클래스**를 기반으로
  **객체(인스턴스)를 생성**해야 한다.
* `new Student()`

    * `new` : **새로 생성한다**는 뜻이다.
    * `new Student()` : `Student` 클래스 정보를 기반으로
      **새로운 객체를 생성하라**는 의미이다.
    * 이렇게 하면 메모리에 실제 `Student` **객체(인스턴스)** 가 만들어진다.
* 객체를 생성할 때는 다음 형식을 사용한다.

    * `new 클래스명()`  → 마지막에 `()` 도 반드시 붙는다.
* `Student` 클래스는 다음 멤버 변수를 가진다.

    * `String name`
    * `int age`
    * `int grade`
      이 변수들을 사용하는 데 필요한 **메모리 공간도 함께 확보**된다.

### 5-3. 3단계: 참조값 보관

<img src="./images/06/03StoreReference.png" width=600px>

객체를 생성하면, 자바는 메모리 어딘가에 있는 이 객체에 접근할 수 있는
**참조값(주소)** 을 반환한다.

* 예시로 이 참조값을 `x001`이라고 표현하자.
  (실제로 `x001`처럼 보이진 않고, 이해를 돕기 위한 예시일 뿐이다.)
* `new` 키워드로 객체가 생성되면, 이 **참조값**이 반환된다.
* 앞에서 선언한 변수 `Student student1`에
  생성된 객체의 참조값 `x001`을 보관한다.

정리하면:

* `Student student1` 변수는 이제 메모리에 존재하는
  실제 `Student` **객체(인스턴스)의 참조값**을 가지고 있다.
* `student1` 변수는 **방금 만든 객체에 접근할 수 있는 열쇠**이다.
  이 변수를 통해 객체를 **접근(참조)** 하고 사용할 수 있다.
  → 즉, `student1`을 통해 메모리에 있는 실제 객체를 접근해서 사용한다.

### 5-4. 참조값을 변수에 보관해야 하는 이유

* `new Student()` 코드 자체에는 **이름이 없다.**

    * 이 코드는 단지 `Student` 클래스를 기반으로
      메모리에 실제 객체를 만드는 역할만 한다.
* 생성된 객체에 접근하려면 **어떤 방법으로든 그 위치(참조값)를 기억**해야 한다.
* 그래서 **객체를 생성할 때 반환되는 참조값을 변수에 보관**한다.

    * 앞에서 `Student student1` 변수에 참조값 `x001`을 저장해 두었다.
    * 이제 저장된 참조값 `x001`을 통해
      실제 메모리에 존재하는 객체에 접근할 수 있다.

지금까지 내용을 간단히 코드 형태로 풀어 쓰면 다음과 같다.

```java
Student student1 = new Student(); // 1. Student 객체 생성
Student student1 = x001;          // 2. new Student()의 결과로 x001 참조값 반환
student1 = x001;                  // 3. 최종적으로 변수 student1에 참조값 보관
```

> 위 코드는 실제 컴파일용이라기보다,
> **참조값이 변수에 저장되는 과정을 단계별로 표현한 것**이다.

이후에 학생2(`student2`)까지 생성하면,
`Student` 객체(인스턴스)가 메모리에 **2개** 생성된다.
각각의 **참조값이 다르기 때문에 서로를 구분**할 수 있다.

### 5-5. 참조값 확인하기

객체의 참조값을 확인하고 싶다면,
객체를 담고 있는 변수를 출력해 보면 된다.

<img src="./images/06/04Reference.png" width=400px>

```java
System.out.println(student1);
System.out.println(student2);
```

출력 예시는 다음과 같다.

```text
class1.Student@7a81197d
class1.Student@2f2c9b19
```

* `@` 앞 부분: `class1.Student`

    * **패키지 + 클래스 정보**를 뜻한다.
* `@` 뒤의 16진수 숫자:

    * 실제 참조값(주소)을 16진수로 표현한 것이다.
