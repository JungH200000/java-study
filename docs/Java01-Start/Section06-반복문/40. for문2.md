# for문 2

**정의**: `for`문은 **초기식→조건식→증감식**을 한 줄에 모아 **반복을 제어**하는 구문이다.
**요약**: 세 부분은 **선택적**이며, 모두 생략하면 **무한 반복**이 된다.

---

## 1) 기본 구조와 생략 형태

```java
for (초기식; 조건식; 증감식) {
    // 코드
}
```

세 부분은 **모두 생략 가능**하다. 단, **세미콜론(;)은 유지**해야 한다.

```java
for (;;) {
    // 코드 (조건 없음 → 무한 반복)
}
```

위 형태는 아래와 **동일한 의미**다.

```java
while (true) {
    // 코드
}
```

---

## 2) 문제: 합계가 10보다 큰 **처음 값** 찾기

> 1부터 시작하여 숫자를 계속 누적해서 더하다가 합계가 **10보다 처음으로 큰 시점의 i와 sum**을 출력하라.

### 예제: `Break2` — `for(;;)` 무한 반복에서 `break`로 탈출

```java
package loop;

public class Break2 {
    public static void main(String[] args) {
        int sum = 0;
        int i = 1;

        for (;;) {                         // 조건 없음 → 무한 반복
            sum += i;                      // 누적
            if (sum > 10) {                // 합이 10보다 커지는 "처음" 순간
                System.out.println("합이 10보다 크면 종료: i=" + i + " sum=" + sum);
                break;                     // 즉시 for문 종료
            }
            i++;                           // 다음 수로 이동
        }
    }
}
```

#### 실행 결과

```
합이 10보다 크면 종료: i=5 sum=15
```

> `for(;;)`는 **무한 루프**이므로, 종료 지점은 코드 내부의 **`break`** 로 제어한다.

## 3) 개선: 증가 변수를 **for 머리**로 옮겨 가독성 높이기

위 코드는 어떤 값이 반복 제어에 쓰이는지 즉시 눈에 들어오지 않는다.
`i`가 **카운터 변수**이므로, 이를 `for`의 **초기식/증감식**으로 옮기면 더 **깔끔**해진다.

### 예제: `Break3` — 카운터를 `for`에 포함

```java
package loop;

public class Break3 {
    public static void main(String[] args) {
        int sum = 0;

        for (int i = 1; ; i++) {           // i 선언(초기식)과 증가(증감식)를 한 줄에
            sum += i;                      // 누적
            if (sum > 10) {                // 종료 조건 충족 시
                System.out.println("합이 10보다 크면 종료: i=" + i + " sum=" + sum);
                break;                     // 즉시 종료
            }
        }
    }
}
```

* `for (int i = 1; ; i++)` 덕분에 **루프 제어 코드(초기화·증가)** 와 **업무 로직(누적·출력)**
* 이 **분리**되어 읽기 쉽다.

---

## 4) 정리

* `while`만으로도 모든 반복을 작성할 수 있다.
* 하지만 **카운터 변수가 명확**하거나 **반복 횟수가 정해진 경우**에는 `for`문이 **구조적으로 더 깔끔**하고 **유지보수**에 유리하다.

> 참고: `for`문을 더 편리하게 사용하는 **향상된 for문(= for-each)** 도 있다. 이 부분은 **뒤에서** 설명한다.
