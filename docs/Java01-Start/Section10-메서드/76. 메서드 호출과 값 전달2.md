# 메서드 호출과 값 전달2

## 1. 메서드 호출과 이름이 같은 변수

이번에는 **호출자의 변수 이름**과 **메서드 매개변수 이름**을 **같게** 해서 같은 문제를 다시 한 번 풀어 보자.

### `MethodValue2` 코드

```java
package method;

public class MethodValue2 {

    public static void main(String[] args) {
        int number = 5;
        System.out.println("1. changeNumber 호출 전, number: " + number); // 출력: 5

        changeNumber(number);

        System.out.println("4. changeNumber 호출 후, number: " + number); // 출력: 5
    }

    public static void changeNumber(int number) {
        System.out.println("2. changeNumber 변경 전, number: " + number); // 출력: 5
        number = number * 2;
        System.out.println("3. changeNumber 변경 후, number: " + number); // 출력: 10
    }
}
```

이번에는 `main()`에 정의한 변수와 메서드의 매개변수(파라미터)의 이름이 **둘 다 `number`로 같다.**

### 실행 결과

```text
1. changeNumber 호출 전, number: 5
2. changeNumber 변경 전, number: 5
3. changeNumber 변경 후, number: 10
4. changeNumber 호출 후, number: 5
```

### `main()`의 `number`와 `changeNumber()`의 `number`

* `main()`도 사실은 **메서드**이다.
* 각 메서드 안에서 사용하는 변수는 **서로 완전히 분리된 다른 변수**이다.
* 변수의 **이름이 같아도** 완전히 다른 변수다.

따라서,

> `main()`의 `number` 와 `changeNumber()`의 `number` 는 **서로 다른 변수**이다.

## 2. 실행 과정 자세히 보기 (`MethodValue2`)

실행 과정을 단계별로 따라가 보자.

```java
changeNumber(number);   // changeNumber를 호출한다. main의 number(5)
changeNumber(5);        // main의 number 값을 읽는다.
// main의 number 값 5가 changeNumber의 number에 복사된다.
// 결과: main의 number(5), changeNumber의 number(5)

void changeNumber(int number = 5) {
    number = number * 2; // changeNumber의 number에 2를 곱한 값을 대입한다.
                         // 결과: main의 number(5), changeNumber의 number(10)
}

main의 number를 출력한다: main의 number의 값인 5가 출력된다.
```

정리하면,

* `main`에서 `number`의 값 5를 읽어서 **복사**한 뒤,
  `changeNumber`의 매개변수 `number`에 전달한다.
* `changeNumber` 안에서 변경되는 것은 **그 메서드의 `number`** 이고,
  `main`의 `number`는 **변경되지 않는다.**

## 3. 메서드 호출과 값 반환받기

그렇다면, **메서드를 사용해서 값을 실제로 변경**하려면 어떻게 해야 할까?

> 메서드의 **호출 결과를 반환** 받아서 사용하면 된다.

### `MethodValue3` 코드

```java
package method;

public class MethodValue3 {

    public static void main(String[] args) {
        int num1 = 5;
        System.out.println("changeNumber 호출 전, num1: " + num1); // 출력: 5

        num1 = changeNumber(num1);

        System.out.println("changeNumber 호출 후, num1: " + num1); // 출력: 10
    }

    public static int changeNumber(int num2) {
        num2 = num2 * 2;
        return num2;
    }
}
```

### 실행 결과

```text
changeNumber 호출 전, num1: 5
changeNumber 호출 후, num1: 10
```

---

## 4. 실행 과정 자세히 보기 (`MethodValue3`)

```java
num1 = changeNumber(num1); // num1(5)
num1 = changeNumber(5);    // num1의 값을 읽는다.

// 호출 시작: changeNumber()
// num1의 값 5가 num2에 대입된다. num1의 값을 num2에 복사한다.
// 결과: num1(5), num2(5)
int changeNumber(int num2 = 5) {
    num2 = num2 * 2; // 계산 결과: num1(5), num2(10)
    return num2;     // num2의 값은 10이다.
}
// 호출 끝: changeNumber()

return 10;
// changeNumber(5)의 반환 결과가 10이다.

num1 = changeNumber(5); // 반환 결과가 10이다.
num1 = 10;              // 결과: num1(10)
```

정리하면,

* 메서드 안에서는 매개변수 `num2`만 바뀐다.
* 메서드가 **`return`을 통해 바뀐 값을 반환**하고,
* 호출한 쪽에서

  ```java
  num1 = changeNumber(num1);
  ```

  처럼 **반환된 값을 다시 대입**해야, 결국 `num1`의 값이 변경된다.
