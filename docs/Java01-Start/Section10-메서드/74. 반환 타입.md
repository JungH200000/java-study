# 반환 타입

## 1. 반환 타입이 있는 메서드의 규칙

**반환 타입(return type)** 이 있는 메서드는 **반드시 값을 반환해야 한다.**
즉, 반환 타입이 `boolean`, `int`, `String` 등이라면,
메서드 안에서 **`return` 문을 통해 해당 타입의 값을 꼭 돌려줘야 한다.**

이 부분은 특히 **`if` 같은 조건문과 함께 사용할 때** 조심해야 한다.

---

## 2. `return` 문을 놓쳤을 때 생기는 문제 – `MethodReturn1`

### `MethodReturn1` (오류 발생 코드)

```java
package method;

public class MethodReturn1 {

    public static void main(String[] args) {
        boolean result = odd(2);
        System.out.println(result);
    }

    public static boolean odd(int i) {
        if (i % 2 == 1) {
            return true;
        }
    }
}
```

이 코드에서:

* `i % 2 == 1` 이면 `return true;` 가 실행된다.
* 하지만 **조건을 만족하지 않으면(`i`가 짝수이면)** `return` 문이 전혀 실행되지 않는다.

즉, 자바 입장에서는

> “`boolean` 을 반환한다고 해 놓고,
> 어떤 경우에는 **아무 것도 반환하지 않을 수도 있다**”

라는 코드가 되어 버린다.

그래서 이 코드를 컴파일하면, 다음과 같은 **컴파일 오류**가 발생한다.

```text
java: missing return statement
```

* 의미: “`return` 문이 빠져 있다”는 뜻이다.

---

## 3. 모든 경우에 `return` 하도록 수정 – `MethodReturn1` 수정 코드

### `MethodReturn1` – 수정 코드

```java
package method;

public class MethodReturn1 {

    public static void main(String[] args) {
        boolean result = odd(2);
        System.out.println(result);
    }

    public static boolean odd(int i) {
        if (i % 2 == 1) {
            return true;
        } else {
            return false;
        }
    }
}
```

이렇게 수정하면:

* `i % 2 == 1` 이면 `if` 블록의 `return true;`가 실행되고,
* 그렇지 않으면(짝수이면) `else` 블록의 `return false;`가 실행된다.

즉, **어떤 경우에도 반드시 `boolean` 값을 반환**하게 되므로
더 이상 `missing return statement` 오류가 발생하지 않는다.

> 정리: **반환 타입이 있는 메서드**는
> **모든 실행 경로에서 `return` 문이 실행되도록** 작성해야 한다.

## 4. `return` 을 만나면 즉시 메서드를 빠져나간다 – `MethodReturn2`

`return` 문을 만나면 그 즉시 **해당 메서드를 빠져나간다.**
다음 로직을 수행하는 메서드를 예제로 보자.

* 18살 미만: `"미성년자는 출입이 불가능합니다."`
* 18살 이상: `"입장하세요."`

### `MethodReturn2` 코드

```java
package method;

public class MethodReturn2 {

    public static void main(String[] args) {
        checkAge(10);
        checkAge(20);
    }

    public static void checkAge(int age) {
        if (age < 18) {
            System.out.println(age + "살, 미성년자는 출입이 불가능합니다.");
            return;
        }
        System.out.println(age + "살, 입장하세요.");
    }
}
```

동작을 정리하면:

* **18세 미만인 경우**

    * `"미성년자는 출입이 불가능합니다."` 를 출력한다.
    * 바로 `return;` 을 만나므로, **다음 줄을 실행하지 않고** 메서드를 빠져나간다.

* **18세 이상인 경우**

    * `if (age < 18)` 조건을 만족하지 않으므로 `if` 블록을 건너뛴다.
    * `"입장하세요."` 를 출력하고, 메서드가 종료된다.
    * 참고로 **반환 타입이 없는 `void` 형**이기 때문에,
      마지막 줄에 `return;` 을 **명시적으로 쓰지 않아도 된다.**

## 5. 반환 값 무시

반환 타입이 있는 메서드를 호출했을 때,
반환된 값이 필요 없다면 **그 값을 사용하지 않고 버려도 된다.**

* 예시1: 반환 값을 변수에 저장

  ```java
  int sum = add(1, 2);  // 반환된 값을 받아서 sum 에 저장했다.
  ```

* 예시2: 반환 값을 사용하지 않음

  ```java
  add(1, 2);  // 반환된 값을 사용하지 않고 버린다.
  ```

여기서는 예시1과 달리, 호출 결과를 변수에 담지 않고
단순히 **메서드만 호출**했다.
즉, **반환 값을 쓰지 않고 버린 것**이다.
