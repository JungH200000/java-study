# 메서드 시작

두 숫자를 입력 받아서 더하고 출력하는 **아주 단순한 기능**을 개발해 보자.
먼저 `1 + 2`를 수행하고, 그 다음으로 `10 + 20`을 수행한다.

---

## 1. 같은 연산을 두 번 수행하는 코드

### `Method1` 예제

```java
package method;

public class Method1 {
    public static void main(String[] args) {
        // 계산1
        int a = 1;
        int b = 2;
        System.out.println(a + "+" + b + " 연산 수행");
        int sum1 = a + b;
        System.out.println("결과1 출력:" + sum1);

        // 계산2
        int x = 10;
        int y = 20;
        System.out.println(x + "+" + y + " 연산 수행");
        int sum2 = x + y;
        System.out.println("결과2 출력:" + sum2);
    }
}
```

* 위 코드는 **같은 종류의 연산(두 수를 더하고 결과 출력)** 을 **두 번** 수행한다.
* 첫 번째는 `1 + 2`, 두 번째는 `10 + 20`을 계산한다.

---

## 2. 계산1과 계산2 비교

코드를 잘 보면 **계산1** 부분과 **계산2** 부분이 거의 같다.

### 계산1 코드

```java
int a = 1;
int b = 2;
System.out.println(a + "+" + b + " 연산 수행");
int sum1 = a + b;
```

### 계산2 코드

```java
int x = 10;
int y = 20;
System.out.println(x + "+" + y + " 연산 수행");
int sum2 = x + y;
```

계산1, 계산2 둘 다 다음과 같은 일을 한다.

1. **변수를 두 개 선언**한다. (`a`, `b` / `x`, `y`)
2. **어떤 연산을 수행하는지 출력**한다. (`System.out.println(...)`)
3. **두 변수를 더해서 결과를 구한다.** (`sum1 = a + b`, `sum2 = x + y`)

즉, **형식은 같고 숫자만 다를 뿐인 코드**가 두 군데에 반복되고 있다.

---

## 3. 같은 코드를 반복해서 작성할 때의 문제

만약 프로그램의 여러 곳에서 이와 같은 계산을 반복해야 한다면 어떻게 될까?

* 같은 코드를 **여러 번 반복해서 작성**해야 한다.
* 더 나아가서
  “어떤 연산을 수행하는지 출력하는 부분”을

    * **변경**하거나
    * **제거**하고 싶다면
      이 코드들을 **모두 찾아다니면서 하나하나 수정**해야 한다.

이런 반복과 수정은 **번거롭고 실수하기 쉽다.**
그래서 이런 문제를 어떻게 깔끔하게 해결할 수 있을지 고민하게 된다.

---

## 4. 수학의 함수(Function)로 생각해 보기

이 문제를 이해하기 위해 잠깐, **아주 간단한 수학의 함수**를 떠올려 보자.

> 함수(function)라는 수학 용어가 나왔다고 해서
> 전혀 어렵게 생각할 필요는 없다.

여기서는 **숫자를 2개 입력**하면
그 숫자를 **더한 다음 결과를 출력하는 아주 단순한 함수**를 생각해 보자.
이 함수의 이름은 `add` 이다.

### 4.1 `add` 함수 정의

함수 정의:

```text
add(a, b) = a + b
```

* 이름이 `add`인 함수이다.
* `a`, `b`라는 **두 값을 입력으로 받는 함수**이다.
* 이 함수는 `a + b` 연산을 수행한다.

### 4.2 `add` 함수 사용

함수 사용 예:

```text
add(1,2) -> 결과:3
add(5,6) -> 결과:11
add(3,5) -> 결과:8
```

* 함수에 **값을 입력하면**,
  함수가 가진 **연산을 처리한 다음 결과를 출력**한다.
* 여기서는 단순히 `a + b` 라는 연산을 수행한다.

여러 번 같은 계산을 해야 한다면,
지금처럼 함수를 **한 번 만들어 두고(정의)**,
필요한 **입력 값을 넣어서 해당 함수를 호출**하면 된다.
그러면 **계산된 결과가 나온다.**

---

## 5. 함수는 블랙박스처럼 동작한다

> 함수는 마치 블랙박스와 같다.

이 말은 다음과 같은 뜻을 담고 있다.

* 함수를 호출할 때, **외부에서는 필요한 값만 입력**하면 된다.
* 그러면 함수 내부에서 정해진 연산을 수행하고,
  **계산된 결과가 출력**된다.
* **같은 함수를 다른 입력 값으로 여러 번 호출**할 수 있다.

여기서 핵심은 다음 한 줄로 정리할 수 있다.

> 함수를 **한 번 정의**해 두면, **계속해서 재사용**할 수 있다!

---

## 6. 평균을 구하는 함수 예시

이번에는 **두 수의 평균**을 구해야 하는 상황을 떠올려 보자.

* 매번 직접 `(a + b) / 2` 라는 공식을 사용해야 한다.
* 이것을 함수로 만들어 두면 더 편리하게 사용할 수 있다.

### 6.1 평균 함수 정의

함수 정의:

```text
avg(a, b) = (a + b) / 2
```

* `a`, `b`라는 두 값을 받아서
* `(a + b) / 2`라는 연산을 수행한다.

### 6.2 평균 함수 사용

함수 사용 예:

```text
avg(4,6)   -> 결과:5
avg(10,20) -> 결과:15
avg(100,200) -> 결과:150
```

이처럼 평균도 **함수로 정의해 두고**,
필요할 때마다 **입력 값만 바꾸어 여러 번 호출**해서 사용할 수 있다.

---

## 7. 수학의 함수 개념과 프로그래밍

정리해 보면 다음과 같다.

* 수학의 **함수 개념을 프로그래밍에 그대로 가져오면**,

    * 필요한 기능을 **미리 정의해 두고**
    * 필요할 때마다 **호출해서 사용**할 수 있다.
* 이렇게 하면 앞에서 본 것처럼

    * 같은 코드를 여러 번 반복해서 작성해야 하는 문제,
    * 여러 곳의 코드를 일일이 찾아가며 수정해야 하는 문제를
      **해결할 수 있을 것 같다.**

> 프로그램 언어들은 오래 전부터 이런 문제를 해결하기 위해
> 수학의 함수라는 개념을 차용해서 사용한다.

즉, **프로그래밍 언어들은 수학의 함수를 모델로 삼아 기능을 재사용**할 수 있게 해 준다.
