# 메서드 사용

자바에서는 **함수**를 **메서드(Method)** 라고 부른다.
메서드는 **함수의 한 종류**라고 생각하면 된다.
지금은 둘을 굳이 구분하지 않고, “메서드 = 함수” 정도로 이해해 두면 충분하다.

앞에서 살펴본 **중복된 계산 코드**를 메서드를 사용해서 **한 번에 깔끔하게 정리**해 보자.
아래 코드는 앞에서 작성한 코드와 **완전히 동일하게 작동**하지만, 메서드를 사용해 중복을 제거한 버전이다.

---

## 1. 메서드로 중복 제거: `Method1Ref`

### `Method1Ref` 코드

```java
package method;

public class Method1Ref {
    public static void main(String[] args) {
        int sum1 = add(5, 10); // add 메서드를 사용해서 5와 10을 더한다.
        System.out.println("결과1 출력:" + sum1);

        int sum2 = add(15, 20); // add 메서드를 사용해서 15와 20을 더한다.
        System.out.println("결과2 출력:" + sum2);
    }

    // add 메서드
    public static int add(int a, int b) {
        System.out.println(a + "+" + b + " 연산 수행");
        int sum = a + b;
        return sum; // 계산 결과를 호출한 쪽으로 반환한다.
    }
}
```

### 실행 결과

```text
5+10 연산 수행
결과1 출력:15
15+20 연산 수행
결과2 출력:35
```

이전 코드에서는 **같은 패턴의 계산 코드**를 여러 번 직접 작성했지만,
이제는 `add`라는 **하나의 메서드**를 정의해 두고,
필요할 때마다 **다른 입력 값으로 호출**해서 사용하고 있다.

> 중복이 제거되고, 코드가 훨씬 깔끔해진 것을 느낄 수 있을 것이다.

---

## 2. 메서드 정의: `add` 메서드 뜯어 보기

### 메서드 전체 코드

```java
public static int add(int a, int b) {
    System.out.println(a + "+" + b + " 연산 수행");
    int sum = a + b;
    return sum;
}
```

이 부분이 바로 **메서드 정의**이다.
수학에서 **함수를 정의하는 것**과 비슷하게, 여기서는 **메서드를 정의한다**고 표현한다.

메서드는 **수학의 함수와 유사**하게 생겼다.

* 함수에 **값을 입력**하면,
* 어떤 **연산을 처리**한 다음,
* **결과를 반환**한다.

> (수학 자체에 너무 집중할 필요는 없다.
> 단순히 “무언가를 미리 정의해 두고, 필요할 때 불러서 사용하는 것”으로 이해해도 충분하다.)

메서드는 크게 두 부분으로 나눌 수 있다.

1. **메서드 선언(Method Declaration)**
2. **메서드 본문(Method Body)**

---

## 3. 메서드 선언 (Method Declaration)

```java
public static int add(int a, int b)
```

이 줄이 **메서드 선언부**이다.
메서드 선언에는 다음 정보가 들어 있다.

* 메서드 이름
* 반환 타입
* 매개변수(파라미터) 목록

이름 그대로, “이런 메서드가 있다”고 **선언**하는 부분이다.
다른 코드에서는 이 선언 정보를 보고,
해당 메서드를 **호출해서 사용할 수 있다.**

### 3-1. `public static`

```java
public static
```

* `public`
  : **다른 클래스에서 호출할 수 있는 메서드**라는 뜻이다.
  (접근 제어에 대한 자세한 내용은 뒤에서 학습한다.)

* `static`
  : **객체를 생성하지 않고도 호출할 수 있는 정적 메서드**라는 뜻이다.
  (자세한 내용은 뒤에서 다룬다.)

두 키워드의 자세한 내용은 뒤에서 다시 학습할 예정이다.
지금은 단순히 "**메서드를 만들 때 이런 키워드를 함께 사용한다**" 정도로 이해하면 된다.

### 3-2. `int add(int a, int b)`

```java
int add(int a, int b)
```

* `int`
  : **반환 타입**을 정의한다.
  메서드가 실행을 마친 뒤 **반환할 결과의 타입**을 지정한다.

* `add`
  : 메서드의 **이름**이다.
  이 이름으로 메서드를 **호출**할 수 있다.

* `(int a, int b)`
  : 메서드를 호출할 때 넘겨줄 **입력 값(매개변수)** 을 정의한다.
  이 변수들은 **해당 메서드 안에서만 사용**된다.

이렇게 메서드 선언에 사용되는 변수를

* 영어로 **파라미터(parameter)**
* 한글로 **매개변수**

라고 부른다.

---

## 4. 메서드 본문 (Method Body)

### 메서드 본문의 형태

```java
{
    System.out.println(a + "+" + b + " 연산 수행");
    int sum = a + b;
    return sum;
}
```

중괄호 `{ ... }` 안이 메서드가 실제로 **수행해야 하는 코드 블록**이다.

* 메서드를 호출하면,
* 메서드 본문에 있는 코드들이 **위에서 아래로 순서대로 실행**된다.

또한 메서드 본문은 **블랙박스**처럼 볼 수 있다.

* 메서드를 사용하는 쪽에서는
  **메서드 선언(이름, 매개변수, 반환 타입)** 은 알지만,
* 메서드 안에서 **어떻게 계산하는지(본문 내용)** 은 몰라도 된다.

### `return` 문

메서드의 실행 결과를 호출한 쪽으로 돌려주려면 **`return` 문**을 사용한다.
`return` 뒤에 **반환할 값**을 적어 주면 된다.

```java
return sum;
```

* `sum` 변수 안에 들어 있는 값을 **반환**한다.
* 이 값이 메서드를 호출한 코드 쪽으로 **전달**된다.

## 5. 메서드 호출

앞서 정의한 메서드를 실제로 사용하려면,
**메서드 이름에 입력 값을 전달하면서 호출**하면 된다.
이를 보통 **“메서드를 호출(call)한다”** 라고 표현한다.

### 5-1. 기본 호출 예

```java
int sum1 = add(5, 10);
int sum2 = add(15, 20);
```

* `add(5, 10)` : `a = 5`, `b = 10`으로 계산해서 결과를 반환
* `add(15, 20)` : `a = 15`, `b = 20`으로 계산해서 결과를 반환

### 5-2. 단순 치환 관점에서 보기

```java
int sum1 = add(5, 10); // add라는 메서드를 숫자 5, 10을 전달하면서 호출한다.
int sum1 = 15;         // add(5, 10)이 실행되고, 반환 값 15가 들어온 상태라고 볼 수 있다.
// sum1에 15 값이 저장된다.
```

메서드를 호출하면 메서드는 계산을 끝내고 **결과를 반환**한다.
쉽게 말해서,

> 메서드 호출이 끝나면,
> 그 위치는 **메서드가 반환한 결과 값으로 치환된 것처럼** 생각할 수 있다.

---

## 6. 메서드 호출 흐름을 단계별로 이해하기

조금 더 자세히 메서드 호출 과정을 살펴보자.
(아래 코드는 **실제 실행용이 아니라, 흐름을 설명하기 위한 예시**라고 보면 된다.)

```java
// 1: 메서드 호출
int sum1 = add(5, 10);

// 2: 파라미터 변수 a=5, b=10이 전달되면서 메서드가 수행된다.
public static int add(int a, int b) { // a = 5, b = 10
    int sum = a + b;
    return sum;
}

// 3: 메서드가 수행된다. 실제 연산은 다음과 같이 진행된다.
public static int add(int a, int b) { // a = 5, b = 10
    int sum = 5 + 10;
    return sum;
}

// 4: return을 사용해서 메서드 실행의 결과인 sum을 반환한다.
// sum에는 값 15가 들어 있으므로, 값 15가 반환된다.
public static int add(int a, int b) { // a = 5, b = 10
    int sum = 15;
    return sum; // 15 반환
}

// 5: 메서드 호출 결과로 메서드에서 반환한 값 15가 나온다.
// 이 값을 sum1에 대입했다.
int sum1 = 15;
```

정리하면,

1. `add(5, 10)` 을 호출한다.
2. 메서드 안으로 **`a = 5`, `b = 10`** 이 전달된다.
3. 메서드 본문에서 `a + b` 연산을 수행한다.
4. `return sum;` 을 통해 결과 `15`를 반환한다.
5. 호출한 곳에서는 `add(5, 10)` 자리에 **반환된 값 15가 온 것처럼** 사용된다.

### 6-1. 메서드 호출 후 메모리 정리

메서드 호출이 끝나면, 더 이상 그 메서드에서 사용한 **임시 변수들을 유지할 필요가 없다.**

따라서 메서드 호출이 끝나면:

* 메서드 정의에 사용한 **파라미터 변수**
  `int a`, `int b`
* 메서드 안에서 정의한 **지역 변수**
  `int sum`

과 같은 것들은 모두 **제거**된다.

---

## 7. 메서드 호출과 용어 정리

메서드를 호출할 때는 **메서드에 넘기는 값**과
메서드가 정의한 **매개변수(파라미터)의 타입이 서로 일치**해야 한다.

또한:

* **순서**
* **개수**

도 맞아야 한다.

예를 들어:

```text
호출: call("hello", 20)
메서드 정의: int call(String str, int age)
```

* `"hello"` → `String str`에 전달
* `20` → `int age`에 전달

---

## 8. 인수(Argument)와 매개변수(Parameter)

### 8-1. 인수(Argument)

> 여기서 `"hello"`, `20`처럼 **메서드에 넘기는 값**을
> 영어로 **Argument(아규먼트)**,
> 한글로 **인수** 또는 **인자**라고 부른다.

실무에서는

* **아규먼트**
* **인수**
* **인자**

라는 용어를 모두 사용한다.

### 8-2. 매개변수(Parameter)

> 메서드를 정의할 때 선언한 변수인
> `String str`, `int age` 를
> **매개변수**, **파라미터(parameter)** 라고 한다.

메서드를 호출할 때 **인수(Argument)** 를 넘기면,
그 인수 값이 **매개변수(Parameter)** 에 대입된다.

실무에서는

* **매개변수**
* **파라미터**

라는 용어를 모두 사용한다.

---

## 9. 용어 정리 (의미)

* **인수(Argument)**

    * ‘인’과 ‘수’의 합성어로,
      **‘들어가는 수’** 라는 의미를 가진다.
    * 즉, **메서드 내부로 들어가는 값**을 의미한다.
    * **인자**도 같은 의미이다.

* **매개변수(Parameter)**

    * ‘매개’와 ‘변수’의 합성어로,
      **‘중간에서 전달하는 변수’** 라는 의미를 가진다.
    * 즉, **메서드 호출부와 메서드 내부 사이에서 값을 전달하는 역할을 하는 변수**라는 뜻이다.
