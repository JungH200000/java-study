# 정리

## 변수명 vs 메서드명

* **변수 이름**은 일반적으로 **명사**를 사용한다.
* **메서드 이름**은 무언가를 **동작**시키는 데 사용하기 때문에, 일반적으로 **동사로 시작**한다.
* 이런 차이점 외에는 **변수 이름과 메서드 이름에 대한 규칙은 둘 다 같다.**

예시:

* **변수명 예)**
  `customerName`, `totalSum`, `employeeCount`, `isAvailable`
* **메서드명 예)**
  `printReport()`, `calculateSum()`, `addCustomer()`, `getEmployeeCount()`, `setEmployeeName()`

---

## 메서드 사용의 장점

메서드는 코드를 더 잘 구조화하고 다루기 쉽게 만들어 준다.

### 1. 코드 재사용

* 메서드는 특정 기능을 **캡슐화**한다.
* 동일한 기능이 필요할 때마다 **코드를 다시 작성하지 않고**, 해당 메서드를 **호출해서 재사용**할 수 있다.

### 2. 코드의 가독성

* 이름이 부여된 메서드는 **코드가 수행하는 작업을 명확하게 표현**한다.
* 코드를 읽는 사람에게 **추가적인 문맥**을 제공해 준다.

### 3. 모듈성

* 큰 프로그램을 **작고 관리 가능한 부분**으로 나눌 수 있다.
* 이렇게 나누면 코드의 **가독성이 향상**되고, **디버깅이 쉬워진다.**

### 4. 코드 유지 관리

* 메서드를 사용하면, 코드의 특정 부분에서 **문제가 발생하거나 업데이트가 필요할 때**
  해당 **메서드만 수정**하면 된다.
* 이렇게 하면 **전체 코드 베이스에 영향을 주지 않고** 변경 사항을 적용할 수 있다.

### 5. 재사용성과 확장성

* 잘 설계된 메서드는 **다른 프로그램이나 프로젝트에서도 재사용**할 수 있다.
* 또한 **새로운 기능을 추가하거나 기존 기능을 확장**하는 데도 유용하다.

### 6. 추상화

* 메서드를 사용하는 위치에서는 **메서드의 구현을 몰라도 된다.**
* 프로그램의 다른 부분은 **복잡한 내부 작업을 알 필요 없이** 메서드를 사용할 수 있다.

### 7. 테스트와 디버깅 용이성

* 개별 메서드는 **독립적으로 테스트**하고 **디버그**할 수 있다.
* 이는 코드의 **문제를 신속하게 찾고 수정**하는 데 도움이 된다.

> 따라서, 메서드는 **효율적이고 유지 보수가 가능한 코드**를 작성하는 데 매우 중요한 도구이다.
