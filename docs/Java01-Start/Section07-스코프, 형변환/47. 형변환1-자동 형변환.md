# 형변환 1 — 자동 형변환

> **한 줄 요약:** **작은 범위 → 큰 범위**로의 숫자 대입은 **자동 형변환(묵시적 형변환)** 이 일어나며 안전하게 동작한다.

## 기본 개념

* **형변환(타입 변환)**: 대입 시 **타입을 맞추기 위해** 값의 타입이 바뀌는 것. 예: `(double) 10`
* **자동 형변환(묵시적 형변환)**: **개발자가 명시하지 않아도** 작은 범위 숫자 타입을 **큰 범위 숫자 타입**에 대입할 때 **자동으로** 일어나는 형변환

## 숫자 타입 범위

* 범위 관계: `int < long < double`
  → `int`보다 `long`이, `long`보다 `double`이 **더 큰 범위**를 표현한다. (`double`은 **부동 소수점** 사용)

## 큰 범위 ↔ 작은 범위

* **작은 → 큰**: **허용** (문제 없이 수행)
* **큰 → 작은**: 다음 문제가 **발생할 수 있음**

    * **소수점 버림**
    * **오버플로우**

---

## 예제: 작은 → 큰 대입 (자동 형변환)

### 코드: `Casting1`

```java
package casting;

public class Casting1 {
    public static void main(String[] args) {
        int intValue = 10;
        long longValue;
        double doubleValue;

        longValue = intValue;         // int -> long (자동 형변환)
        System.out.println("longValue = " + longValue); // longValue = 10

        doubleValue = intValue;       // int -> double (자동 형변환)
        System.out.println("doubleValue1 = " + doubleValue); // doubleValue1 = 10.0

        doubleValue = 20L;            // long -> double (자동 형변환)
        System.out.println("doubleValue2 = " + doubleValue); // doubleValue2 = 20.0
    }
}
```

### 실행 결과

```
longValue = 10
doubleValue1 = 10.0
doubleValue2 = 20.0
```

---

## 자동 형변환의 개념적 전개

아래는 **동작 과정을 개념적으로 풀어쓴 것**이다. (실제 코드 흐름을 설명하기 위한 단계 표시)

```java
// intValue = 10
doubleValue = intValue;            // 대입 시 타입 맞추기 필요
doubleValue = (double) intValue;   // (개념) 형을 double로 맞춤
doubleValue = (double) 10;         // (개념) 변수 값 읽기
doubleValue = 10.0;                // (개념) 형변환 결과
```

* 작은 범위 숫자 타입을 큰 범위 숫자 타입에 대입할 때, 위와 같은 **형 맞추기 과정이 자동**으로 일어난다.

---

## 정리

* **작은 범위 → 큰 범위**의 대입은 **허용**되며 자동 형변환이 일어난다.
* **큰 범위 → 작은 범위**는 **소수점 버림**이나 **오버플로우** 위험이 있다.
* 쉽게 말해, **큰 그릇은 작은 그릇의 내용물을 담을 수 있다.**
