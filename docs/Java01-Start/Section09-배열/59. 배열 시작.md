# 배열 시작

## 왜 배열이 필요한가? (문제 배경)

* 학생 점수를 **각각의 변수**로 관리하면, 학생 수가 늘어날수록

    1. **변수 선언**이 같은 패턴으로 끝없이 증가하고
    2. **출력 코드**도 줄마다 직접 추가해야 하며
    3. 변수 **이름이 모두 달라**서 **반복문을 적용하기 어렵다**는 문제가 생깁니다.
* 결론: **같은 타입의 변수를 반복 선언·사용하는 문제를 해결**하는 도구가 **배열**입니다.

> **배열(핵심 정의)**: *같은 타입*의 여러 값을 **하나로 묶어** 다루는 저장 구조.
> **한 줄 요약**: 비슷한 변수를 줄줄이 만들지 않고 **묶어서** 관리한다.

---

## 예제: 학생의 점수를 출력하는 간단한 프로그램

### Array1

```java
package array;

public class Array1 {
    public static void main(String[] args) {
        // 각 학생의 점수를 '별도 변수'로 관리
        int student1 = 90;
        int student2 = 80;
        int student3 = 70;
        int student4 = 60;
        int student5 = 50;

        // 개별 변수명으로 직접 출력 (학생이 늘면 줄도 계속 늘어남)
        System.out.println("학생1 점수: " + student1);
        System.out.println("학생2 점수: " + student2);
        System.out.println("학생3 점수: " + student3);
        System.out.println("학생4 점수: " + student4);
        System.out.println("학생5 점수: " + student5);
    }
}
```

### 실행 결과

```
학생1 점수: 90
학생2 점수: 80
학생3 점수: 70
학생4 점수: 60
학생5 점수: 50
```

> **해설**
>
> * 변수 수 = 학생 수(5명 → 5개, 100명 → 100개)로 **선언량이 선형 증가**합니다.
> * 출력도 **변수 이름에 직접 의존**하므로, 학생이 늘 때마다 **출력 줄을 수작업으로 추가**해야 합니다.
> * 변수명이 모두 달라 `for` 같은 **반복문을 적용할 수 없습니다**

---

## 결론: 배열로 해결

* **배열**은 *같은 타입*의 값을 **하나로 묶어 인덱스로 접근**하도록 해 주어,

    * **선언 중복**과 **출력 코드 증식** 문제를 줄이고,
    * **반복문 적용**이 가능해집니다.
