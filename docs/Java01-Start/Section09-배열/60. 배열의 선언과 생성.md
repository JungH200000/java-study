# 배열의 선언과 생성

배열은 **같은 타입의 변수 여러 개를 하나로 묶어** 다루기 쉽게 만든 구조입니다.
이전 예제를 배열로 바꾸는 과정을 단계적으로 살펴봅니다.

## Array1Ref1

```java
package array;

public class Array1Ref1 {
    public static void main(String[] args) {
        int[] students;          // 배열 변수 선언
        students = new int[5];   // 배열 생성(길이 5)

        // 변수 값 대입
        students[0] = 90;
        students[1] = 80;
        students[2] = 70;
        students[3] = 60;
        students[4] = 50;

        // 변수 값 사용
        System.out.println("학생1 점수: " + students[0]);
        System.out.println("학생2 점수: " + students[1]);
        System.out.println("학생3 점수: " + students[2]);
        System.out.println("학생4 점수: " + students[3]);
        System.out.println("학생5 점수: " + students[4]);
    }
}
```

---

## 두 줄 집중 설명

```java
int[] students;        // 1. 배열 변수 선언
students = new int[5]; // 2. 배열 생성
```

### 1) 배열 변수 선언

* `int[] students;` 처럼 **타입 뒤에 대괄호(`[]`)** 가 붙습니다.
* **배열 변수를 선언했다고 해서 배열 자체가 만들어진 것은 아닙니다.**
* `int a` 에는 정수, `double b` 에는 실수를 담듯이, `int[] students` 같은 **배열 변수에는 ‘배열’** 을 담습니다.
  (즉, `10`, `20` 같은 **단일 값**이 아니라 **배열 그 자체**를 담습니다.)

### 2) 배열 생성

* `new int[5]` 는 **int 형 변수 5개**를 연속으로 만드는 뜻입니다.

    * `new`: 새로 만든다
    * `int[5]`: 길이가 5인 `int` 공간
* 이전에 `int student1` … `int student5` 를 **직접 5번 선언**했던 과정을, **한 번에** 처리합니다.

---

## 배열과 초기화

* `new int[5]` 로 만들면 **총 5개의 int** 공간이 생기고, 자바가 **자동 초기화**합니다.

    * 숫자: `0`
    * `boolean`: `false`
    * `String`: `null` (없음을 의미)

---

## (3) 배열 참조값 보관

* `new int[5]` 로 생성하면, 배열 크기만큼 메모리를 확보합니다.
  예) `int` 5개 → `4byte * 5 = 20byte`
* 자바는 이 배열에 접근할 수 있는 **참조값(주소)**(예: `x001`)을 반환합니다.
* 선언했던 `int[] students` 변수에 **참조값(`x001`)을 보관**합니다. 이 참조값을 통해 **메모리의 실제 배열**을 사용할 수 있습니다.
* 참고: `new int[5]` 자체에는 **이름이 없습니다**. 생성 시 반환된 **참조값을 변수에 저장**해 접근합니다.

```java
// [의사코드] 개념 설명용
int[] students = new int[5]; // 1) 배열 생성 → 참조값 반환(예: x001)
int[] students = x001;       // 2) 반환된 참조값 예시
students = x001;             // 3) 최종적으로 students가 그 배열을 가리킴
```

---

## 참조값 확인해 보기

```java
System.out.println(students);
```

예시 출력:

```
[I@4617c264
```

* `@` 앞의 **`[I`** 는 **int형 배열**을 뜻합니다.
* `@` 뒤의 **16진수**는 **참조값(주소)** 을 의미합니다.
