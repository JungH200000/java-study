# 향상된 for문(Enhanced for loop)

## 개념 소개

앞서 반복문에서 설명하지 않은 내용 하나가 있는데, 바로 **향상된 for문(Enhanced For Loop)** 이다.
향상된 for문을 이해하려면 **배열**을 먼저 알아야 한다.

* 배열의 **각각의 요소를 탐색**한다는 의미로 **for-each문**이라고도 자주 부른다.
* **배열을 사용할 때**, 기존 `for`문보다 **더 편리하게** 사용할 수 있다.

---

## 향상된 for문 정의

> **형식**

```java
for (변수 : 배열 또는 컬렉션) {
    // 배열 또는 컬렉션의 요소를 순회하면서 수행할 작업
}
```

* `:`의 **오른쪽**에는 `numbers`처럼 **탐색할 배열(또는 컬렉션)** 을 둔다.
* `:`의 **왼쪽**에는 `int number`처럼 **반복마다 값을 담을 변수**를 둔다.

---

## 예제 코드: EnhancedFor1

```java
package array;

public class EnhancedFor1 {
    public static void main(String[] args) {
        int[] numbers = {1, 2, 3, 4, 5};

        // 일반 for문
        for (int i = 0; i < numbers.length; ++i) {
            int number = numbers[i];
            System.out.println(number);
        }

        // 향상된 for문, for-each문
        for (int number : numbers) {
            System.out.println(number);
        } // 생성 단축키(IntelliJ): `iter`

        // for-each문을 사용할 수 없는 경우, 증가하는 index 값 필요
        for (int i = 0; i < numbers.length; ++i) {
            System.out.println("number" + i + "번의 결과는: " + numbers[i]);
        }
    }
}
```

---

## 일반 for문

```java
// 일반 for문
for (int i = 0; i < numbers.length; ++i) {
    int number = numbers[i];
    System.out.println(number);
}
```

먼저 **일반 for문**을 보면,

* 배열에 있는 값을 **순서대로 읽어서** `number` 변수에 넣고,
* 그 값을 출력한다.

배열은 **처음부터 끝까지 순서대로** 읽어서 사용하는 경우가 많다.
하지만 배열의 값을 읽으려면:

1. `int i` 와 같은 **인덱스를 탐색할 변수**를 선언해야 하고,
2. `i < numbers.length` 와 같이 **배열의 끝 조건**을 지정해야 하고,
3. 배열의 값을 하나 읽을 때마다 **인덱스를 하나씩 증가**시켜야 한다.

개발자 입장에서는 **단순히 배열을 처음부터 끝까지 탐색**하고 싶은 것뿐인데,
그에 비해 코드가 **번잡**해진다.
그래서 **향상된 for문**이 등장했다.

---

## 향상된 for문

```java
// 향상된 for문, for-each문
for (int number : numbers) {
    System.out.println(number);
}
```

* 앞서 본 **일반 for문과 동일하게 작동**한다.
* 향상된 for문은

    * 배열의 **인덱스를 직접 사용하지 않고**,
    * **종료 조건을 직접 주지 않아도** 된다.
    * 대신, **해당 배열을 처음부터 끝까지** 탐색한다.

동작 흐름:

1. `:` 오른쪽에 `numbers`처럼 **탐색할 배열**을 둔다.
2. `:` 왼쪽에 `int number`처럼, **반복마다 값을 담을 변수**를 선언한다.
3. 배열의 값을 하나 꺼내서 왼쪽 변수 `number`에 담고, `for`문의 본문을 실행한다.
4. `for`문의 끝에 도달하면, **다음 값**을 꺼내서 다시 `number`에 담고 반복 수행한다.
5. `numbers` 배열의 **끝에 도달해 더 이상 값이 없으면**, `for`문이 완전히 종료된다.

정리하면, 향상된 for문은

* 배열의 인덱스를 사용하지 않고도 **배열의 요소를 순회**할 수 있어서,
* 코드가 **간결**하고, **가독성이 좋다.**

---

## 향상된 for문을 사용할 수 없는 경우

그런데 **향상된 for문을 사용하지 못하는 경우**도 있다.

* 향상된 for문 내부에는 **증가하는 인덱스 값이 감추어져** 있기 때문에,
* `int i`와 같은 **증가하는 인덱스 값을 직접 사용해야 하는 경우**에는
  **향상된 for문을 사용할 수 없다.**

```java
// for-each문을 사용할 수 없는 경우, 증가하는 index 값 필요
for (int i = 0; i < numbers.length; ++i) {
    System.out.println("number" + i + "번의 결과는: " + numbers[i]);
}
```

* 이 예제에서는 증가하는 `i` 값을 **직접 출력**해야 하므로,
* 향상된 for문 대신에 **일반 for문**을 사용해야 한다.

물론, 아래와 같이 **억지스럽게** 향상된 for문을 사용할 수도 있다.

```java
int i = 0;
for (int number : numbers) {
    System.out.println("number" + i + "번의 결과는: " + number);
    i++;
}
```

하지만 이런 경우에는 **일반 for문을 사용하는 것이 더 좋다**고 설명하고 있다.
