# 배열 사용

## 인덱스(Index)

> **정의:** 배열 요소의 **위치 번호**.
> **한 문장 요약:** `[]` 안에 넣는 **0부터 시작하는 번호**다.

```java
// 변수 값 대입
students[0] = 90;
students[1] = 80;

// 변수 값 사용
System.out.println("학생1 점수: " + students[0]);
System.out.println("학생2 점수: " + students[1]);
```

* `students[번호]` 형태로 **특정 위치의 값**을 읽거나 쓴다.
* `[]` 안의 숫자가 바로 **인덱스**다.

---

## 배열은 0부터 시작한다

* `new int[5]` 처럼 **5개의 요소**를 만들면 **인덱스는 `0,1,2,3,4`** 가 된다.
* 사용 가능한 인덱스 범위는 **`0 ~ (n-1)`** 이다. 따라서 **마지막 요소는 `students[4]`**.

인덱스 범위를 넘기면 예외가 발생한다:

```
Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: Index 5 out of bounds for length 5
    at array.Array1Ref1.main(Array1Ref1.java:14)
```

## 배열에 값 대입 (참조 개념 포함)

```java
// 1) 배열에 값을 대입
students[0] = 90;
// 2) [의사코드] 변수에 든 '참조값'을 통해 실제 배열에 접근한다는 개념
x001[0] = 90;

students[1] = 80;
// [의사코드]
x001[1] = 80;
```

> 설명
>
> * 실제 코드는 `students[인덱스]`를 사용한다.
> * `x001` 표기는 **메모리상의 배열 주소(참조값)를 예시로 나타낸 것**이다. (컴파일용 코드 아님)

---

## 배열 값 읽기

```java
// 1) 변수로 읽기
System.out.println("학생1 점수: " + students[0]);

// 2) [의사코드] 참조값을 통해 배열 요소에 접근한다는 개념
System.out.println("학생1 점수: " + x001[0]);

// 3) 최종적으로 읽히는 '값'의 예
System.out.println("학생1 점수: " + 90);
```

> 설명
>
> * 실제 프로그램은 1)처럼 `students[0]`을 통해 값을 읽는다.
> * 2), 3)은 **접근 과정과 결과값을 풀어쓴 개념 설명**이다.

---

## 기본형 vs 참조형

* **기본형(Primitive Type):** `int`, `long`, `double`, `boolean`처럼 **변수 안에 사용할 값을 직접 저장**하는 타입.
* **참조형(Reference Type):** `int[] students`처럼 **데이터가 있는 곳의 참조(주소)를 저장**하는 타입. (뒤에서 학습할 **객체/클래스**를 담는 변수도 모두 참조형)

> 한 문장 요약: **기본형은 값 자체를, 참조형은 값이 있는 곳의 주소를 가진다.**

---

## 참고: 왜 참조형을 쓸까? (동적 크기)

기본형은 크기가 고정되어 있다.

```java
int i;    // 4byte
long l;   // 8byte
double d; // 8byte
```

배열은 **실행 시점에 크기가 정해질 수 있다**.

```java
int size = 10000; // (사용자 입력 등으로 정해졌다고 가정)
new int[size];    // 이 코드를 실행하는 시점에 배열 크기 결정
```

* **기본형**은 선언과 함께 크기가 고정된다.
* **참조형(배열 등)** 은 실행 중 입력 등에 따라 **동적으로 메모리를 할당**할 수 있어 **유연성**을 제공한다.
* 기본형은 값을 직접 저장하므로 **빠르고 메모리 효율적**인 반면, 참조형은 **복잡한 구조**를 만들고 관리할 수 있다.
